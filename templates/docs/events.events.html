Events are things that you can listen for. They can originate from the user like a touch or mouse click, or from an animation that ends. Most objects support event listening in Framer, but you will most often listen for events on layers. When an event is called the first argument is the event information. Depending on the specific event this can contain mouse positions, mouse deltas etc. The second argument is always the layer that the event occured to. To listen for an event you can use the on function: 
layerA = new Layer()
layerA.name = "Layer A"

layerA.on Events.Click, (event, layer) ->
	print "This layer was clicked", layer.name

# Output: "This layer was clicked", "Layer A"
To stop listening for an event you can use the off function: 
layerA = new Layer()
layerA.name = "Layer A"

clickHandler = (event, layer) ->
	print "This layer was clicked", layer.name

layerA.on(Events.Click, clickHandler)
layerA.off(Events.Click, clickHandler)
Framer has a global Events object with events that you can listen to. This is a full overview:
Name Description
Basic
Events.Click Click or touch (no delay on mobile)
Events.TouchStart Start a touch/click
Events.TouchMove Touch move or mouse drag
Events.TouchEnd End touch or click
Mouse
Events.MouseOver Hover mouse
Events.MouseOut Unhover mouse
Animation
Events.AnimationStart Start animation
Events.AnimationStop Stop animation
Events.AnimationEnd End animation (reached end point)
Dragging
Events.DragStart Start drag
Events.DragMove Move drag
Events.DragEnd End drag
Scroll
Events.Scroll Did scroll, does not always work on mobile
States
Events.StateWillSwitch About to switch to a new state
Events.StateDidSwitch Just switched to new state
