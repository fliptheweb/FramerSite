{% endblock sidebar %} {% block body %} {% filter pygmentify %} {% filter markdown %} ### Events Events are things that can happen to a layer, often caused by user interaction. With events, you can animate layers based on these interactions. Framer contains many events: click events, touch events, scroll events, drag events and more. To start listening to an event, write: layer.on event, → ...
To stop listening you can write: layer.off event, → ... 
# Listen to multiple events
layerA.on Events.Click, -> 
    ...
layerA.on Events.TouchStart, -> 
    ...
layerA.on Events.TouchMove, -> 
    ...
layerA.on Events.TouchEnd, -> 
    ...
One of the most common use cases of events is toggling between a set of states on click. 
# Toggle states on click
layerA.on Events.Click, -> 
    imageLayer.states.next()
##### Animation Chaining Events can also be used to chain animations. With these events, you can start a new animation after another one ends. The animation that occurs when switching states also fires an AnimationEnd event. 
# Animation Events
layerA.on Events.AnimationStart, ->
	...
layerA.on Events.AnimationEnd, ->
	...
Below is a simple example of a chained animation. Every animation gets an AnimationEnd event, so they can be chained infinitely. 
layerA.animate 
	properties:
		x:200
	curve: "ease-in-out"
		
layerA.on Events.AnimationEnd, ->
	layerA.animate
		properties:
			x:100
		curve: "ease-in-out"

Summary

Events in a nutshell

  • Events are things that can happen to a layer
  • Events are often used to toggle states
  • Animations can be chained with the AnimationEnd event
