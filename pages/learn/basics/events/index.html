{% endblock sidebar %} {% block body %} {% filter pygmentify %} {% filter markdown %} ### Events Events are things that can happen to a layer, often caused by user interaction. They include a click, mouse over, a scroll and more. To listen to an event, use layer.on event, → ...
To stop listening you can use layer.off event, → ... 
# Listen to multiple events
layerA.on Events.Click, -> 
    ...
layerA.on Events.TouchStart, -> 
    ...
layerA.on Events.TouchMove, -> 
    ...
layerA.on Events.TouchEnd, -> 
    ...
One of the most common uses of Events in Framer is switching between a set of states: 
# Toggle states on click
layerA.on Events.Click, -> 
    imageLayer.states.next()
##### Animation Chaining Events can also be used to chain animations, either via states or layer animations. With these events, you can start a new animation after another one ends. Note that the animation events also work if you've only defined states. The animation that occurs after switching states also fires an AnimationEnd event. 
# Animation Events
layerA.on Events.AnimationStart, ->
	...
layerA.on Events.AnimationEnd, ->
	...

# States Events
layerA.on Events.StateWillSwitch, ->
	...
layerA.on Events.StateDidSwitch, ->
	...
